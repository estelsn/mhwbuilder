Monster Hunter Wilds Set Builder
몬스터 헌터 와일즈 장비 세팅 추천 시스템

## 1. 프로젝트 개요
사용자가 원하는 스킬 조건을 입력하면,
세트 스킬과 장식주 슬롯 제약을 고려하여
실제 게임에서 구현 가능한 장비 조합을 추천하는 백엔드 시스템입니다.

본 프로젝트는 단순 조합 생성이 아니라  
조건을 만족하는지 검증하는 "추천 엔진" 구현에 초점을 맞추었습니다.

## 2. 개발 배경
몬스터 헌터 시리즈에서는 원하는 스킬 조합을 맞추기 위해
  수많은 장비 조합과 장식주 배치를 직접 계산해야 하는 문제가 있습니다.
- 특히 다음과 같은 어려움이 있습니다:

- 세트 스킬 조건 (n개 장착 시 발동)
- 장식주 슬롯 크기 제약
- 스킬 레벨 합산 계산
- 장비 부위 중복 불가

이 프로젝트는 이러한 복잡한 조건을 자동으로 계산하여
#### "실제로 구현 가능한 세팅" 만 추천하는 것을 목표로 개발되었습니다.

초기에는 대규모 프로젝트로 설계했으나 MVP 형태로 방향을 재설정했습니다.


## 3. 기술 스택
- Backend: Spring Boot
- Language: Java 21
- ORM: Spring Data JPA
- DB: H2 (MVP 기준)
- Crawling: Jsoup
- JSON Parsing: Jackson

## 4. 핵심 기능

- 스킬 기반 장비 세팅 추천
    - 일반 스킬 + 세트(그룹, 시리즈) 스킬 동시 선택 가능
- 세트 스킬 조건 충족 조합 탐색
- 장식주 슬롯 기반 실제 장착 가능 여부 검증
- 장식주 자동 배치 및 결과 계산
- 방어력 및 속성 저항 최종 스탯 계산
- 추천 결과 정렬 (슬롯 여유 기반)

## 5. 구조 설계 특징
- 데이터 정규화를 통한 1:N 중심 구조 설계 (M:N 최소화)
- KSUID 기반 ID 생성으로 분산 환경 및 확장성 고려
- DB 교체 가능 구조 설계 (MySQL → H2 전환 경험)
- 크롤링 데이터 → DB 적재 → 추천 로직까지 전체 흐름 직접 설계

## 6. 핵심 설계 포인트

### 1. 세트 스킬 우선 필터링

세트 스킬은 조합 수를 폭발적으로 증가시키는 요소이기 때문에,
초기 단계에서 조건을 만족하는 장비 조합만 생성하도록 설계했습니다.

→ 불필요한 탐색 공간을 크게 줄임


### 2. 장비 조합 → 검증 구조 분리

추천 로직을 다음 단계로 분리했습니다:

1. 세트 스킬 만족 조합 생성
2. 부족 부위 채우기
3. 장식주로 스킬 충족 가능 여부 검증

→ "생성"과 "검증"을 분리하여 로직 단순화


### 3. 장식주 슬롯 검증 알고리즘

장식주는 하위 슬롯 → 상위 슬롯 장착 가능하다는 규칙을 기반으로
다음 조건으로 검증했습니다:

- need3 ≤ slot3
- need2 + need3 ≤ slot2 + slot3
- need1 + need2 + need3 ≤ total slots

→ 실제 게임 규칙을 반영한 검증 로직


### 4. 추천 품질 기준 (슬롯 여유)

단순히 조건을 만족하는 것이 아니라,
"더 여유 있는 세팅"을 상위로 정렬하도록 설계했습니다.

→ 확장 가능성과 실전 활용도 반영


### 5. DTO 기반 응답 설계

프론트엔드와의 분리를 위해
엔티티가 아닌 DTO를 통해 결과를 구성했습니다.

→ 유지보수성과 확장성 확보

## 7. 추천 로직 흐름

1. 사용자 입력
    - 일반 스킬 (skillId + targetLevel)
    - 세트 스킬 (setSkillId + requiredCount)

2. 세트 스킬 조건을 만족하는 장비 조합 생성

3. 남은 장비 부위를 일반 스킬 기준으로 채움

4. 장비 + 호석 기준 현재 스킬 레벨 계산

5. 부족한 스킬을 장식주로 채울 수 있는지 검증

6. 가능한 조합만 필터링

7. 슬롯 여유 기준으로 정렬

8. 최종 결과 반환
